# Shrinkr

Компрессор изображений в трее Windows. Одна картинка через Drag-n-Drop —
сжатый файл рядом с исходником. Правило: минимальнейшие потери качества при
наибольшем сжатии, без изменения разрешения.

## Движки сжатия (как у Caesium / libcaesium)

| Формат | Lossless | Smart (по умолчанию) |
|--------|----------|----------------------|
| JPEG   | mozjpeg: прогрессивный + оптимальный Хаффман | пережатие q87 (сохранение субдискретизации, ICC/EXIF) + mozjpeg |
| PNG    | oxipng   | libimagequant (движок pngquant, порог качества 80–99) + oxipng |
| WebP   | lossless-источник → lossless method=6 | lossy q88 method=6 (принимается только при выигрыше ≥10%) |
| GIF    | gifsicle -O3 | gifsicle -O3 --lossy=30 (при выигрыше ≥10%); фолбэк — Pillow |

Результат принимается только если он меньше исходника; иначе файл не трогается
(«Already optimized»). Пережатие с потерями принимается только при выигрыше
≥10% против безпотерьного результата.

## Запуск

```
pip install -r requirements.txt
python main.py
```

Приложение живёт в трее: ЛКМ — открыть/закрыть окно, ПКМ — меню (Open/Exit).
При первом открытии окна докачивается `gifsicle.exe`
(в `%APPDATA%/Shrinkr/tools`). Конфиг — `%APPDATA%/Shrinkr/config.json`
(кнопка Reset Settings сбрасывает его и перезапускает программу).

## Обновления

Проверка релизов GitHub (SmeshidoJoe/Shrinkr): вручную из окна About
(Check for Updates → Download Update) либо фоново раз в 6 часов с тостом
«Update available» (отключается галочкой Notify about updates в настройках).
Скачанный zip применяется помощником с заменой exe и перезапуском.

## Настройки

- **Mode**: Smart (незаметные потери, максимум сжатия) / Lossless (пиксели не меняются)
- **Quality**: пресеты Max/Balanced/Small + ползунок 60–95 (влияет на Smart)
- **Convert to**: Keep format / WebP / AVIF — пережатие в другой формат
- **Output**: рядом с оригиналом (`name-min.ext`) / перезапись оригинала
- **Strip metadata**: удалять EXIF (камера, GPS) и вспомогательные чанки
- **Explorer context menu**: пункт «Compress with Shrinkr» по ПКМ на картинке
  (второй запуск передаёт файл работающему инстансу)
- Режим окна (Pinned/Auto-hide), перетаскивание, иконка трея, тема, язык,
  уведомления об обновлениях, автозапуск, копия результата в буфер, сброс

## UX

- Превью до/после с размерами на экране результата
- Результат можно вытащить из окна перетаскиванием (drag-out) прямо в
  мессенджер/проводник
- Тёплый процесс-воркер: первый файл жмётся без задержки на запуск процесса;
  интерфейс никогда не блокируется кодеками
